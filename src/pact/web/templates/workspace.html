Get started

This browser keeps the signing key for this registry. If you already used this site here, PACT will unlock it automatically. If not, it creates a new private identity and registers a public profile for you.

Your public profile can be anonymous. The registry receives your public key and a browser fingerprint used to limit duplicate signups; your private key stays in this browser.

Recovery and advanced options

Import an identity made by the CLI, or download a recovery file for this browser identity. A CLI import may ask for the password used when the CLI identity was exported.

Claims and profile

This publishes your public key and device/browser fingerprint to the registry. After that, you can register signed manifests as claims without recreating the identity. Display name is optional; leave it blank to use a pseudonymous key ID only.

Register a signed manifest from the signing page, or upload one that was created earlier.

Revoke one of your claims if it should no longer be trusted.

Sign content

The file is read in your browser. PACT stores a signed manifest and nonce for verification. Register the resulting manifest from the claims page.

Verify and audit

Verification checks signatures and optional content binding. Privacy audit checks what would become public before you register a claim.

Watermarks

Text watermarking edits prose in your browser. Image watermarking uses the same TrustMark-backed Python path as the CLI when the browser runtime can load the required image dependencies.

Document carriers

PDF and ZIP-based documents use the same Python embedding code as the CLI. Native C2PA SDK operations load only when available in the browser runtime.

Model probes

Create probes locally, paste model responses later, and export the evidence package.
