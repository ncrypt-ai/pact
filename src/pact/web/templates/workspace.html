Private keys stay in this browser unless you export them. Registry actions are signed locally before they are submitted.

1. Identity

Create or import the key that signs your claims. Save the encrypted export; losing it means losing the signing identity.

2. Register profile

This publishes your public key and device/browser fingerprint to the registry. The fingerprint is anti-spam friction, not proof of legal identity.

3. Sign and register content

The file is read in your browser. PACT stores a signed manifest and nonce for verification; the registry receives the signed manifest when you register the claim.

4. Verify and audit

Verification checks signatures and optional content binding. Privacy audit checks what would become public before you register a claim.

5. Document carriers

PDF and ZIP-based documents use the same Python embedding code as the CLI. Native C2PA SDK operations load only when available in the browser runtime.

6. Model probes

Create probes locally, paste model responses later, and export the evidence package.
